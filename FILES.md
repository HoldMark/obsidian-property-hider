# Описание файлов проекта

## Исходный код

### `main.ts`
Единственный файл с логикой плагина. Содержит класс плагина, настройки (`MetadataHiderSettings`), генерацию CSS (`genAllCSS`, `updateCSS`), отслеживание фокуса на контейнере метаданных, DOM-манипуляции для панели «All properties», миграцию старых настроек и весь UI настроек.

### `src/i18n.ts`
Локализация строк интерфейса. Содержит переводы на английский (EN) и русский (RU) языки. Класс `Locals.get()` возвращает нужный объект строк в зависимости от языка Obsidian (`localStorage.language`).

### `src/util.ts`
Вспомогательная утилита. Экспортирует единственную функцию `string2list` — преобразует строку с разделителями-запятыми в очищенный массив строк (используется при разборе списков свойств из настроек).

### `styles.css`
Статические CSS-стили плагина. Содержит правило `display: none` для элементов `.tree-item.mh-hide` в панели «All properties» — класс, который плагин навешивает через DOM-манипуляции (в отличие от динамически генерируемых стилей для таблицы свойств, которые инжектируются через `<style>` тег).

## Сборка

### `esbuild.config.mjs`
Конфигурация сборщика esbuild. В режиме разработки (`npm run dev`) запускает `watch` с inline-sourcemaps; в production (`npm run build`) собирает минифицированный бандл без sourcemap. Точка входа — `main.ts`, выход — `main.js`. Obsidian и Node built-ins помечены как `external` (не включаются в бандл).

### `version-bump.mjs`
Скрипт обновления версии (`npm run version`). Читает новую версию из `package.json`, записывает её в `manifest.json` и добавляет запись `{ "версия": minAppVersion }` в `versions.json`. Запускается перед git-тегированием.

### `tsconfig.json`
Конфигурация TypeScript-компилятора. Включает строгие проверки (`noImplicitAny`, `strictNullChecks`), target ES6, inline sourcemaps. Используется при `tsc -noEmit` (проверка типов в `npm run build`) и IDE.

### `Makefile`
Вспомогательные команды для релиза. `make push tag=x.y.z` — ставит git-тег и пушит его в origin (триггер для GitHub Actions). `make del tag=x.y.z` — удаляет тег локально и удалённо.

## Манифесты и метаданные

### `manifest.json`
Основной манифест плагина для Obsidian. Содержит `id`, `name`, `version`, `minAppVersion`, описание и ссылки автора. Obsidian читает этот файл при установке и обновлении плагина.

### `manifest-beta.json`
Манифест для бета-канала (BRAT). Аналогичен `manifest.json`, но используется для тестирования предрелизных версий через плагин BRAT.

### `versions.json`
Таблица совместимости: маппинг `{ "версия плагина": "minAppVersion Obsidian" }`. Нужен Obsidian Community Plugins для проверки, поддерживает ли текущая версия приложения данную версию плагина.

### `package.json`
Конфигурация npm-проекта. Определяет скрипты (`dev`, `build`, `version`), dev-зависимости (esbuild, TypeScript, типы Obsidian, ESLint-плагины).

### `package-lock.json`
Автоматически генерируемый lock-файл npm. Фиксирует точные версии всех транзитивных зависимостей для воспроизводимых сборок.

## Конфигурация инструментов

### `.eslintrc`
Конфигурация ESLint с плагином `@typescript-eslint`. Задаёт правила линтинга: запрет неиспользуемых переменных, разрешение `@ts-comment` и пустых функций.

### `.editorconfig`
Настройки форматирования редактора (отступы табами по 4 пробела, LF-окончания, UTF-8, финальный перевод строки). Применяется в любом редакторе с поддержкой EditorConfig.

### `.eslintignore`
Список файлов и папок, исключённых из проверки ESLint (обычно `node_modules`, собранный `main.js`).

### `.npmrc`
Конфигурация npm. Содержит `tag-version-prefix=""` — убирает префикс `v` при `npm version`, чтобы теги имели вид `1.0.2`, а не `v1.0.2`.

### `tsconfig.json`
Описан выше в разделе «Сборка».
